Memory Filter

記憶寫入前的三層過濾機制

不是所有對話都值得記住 — 三層過濾確保只有有價值的知識進入 LanceDB

→

Layer 1 — Noise Filter
Regex 規則匹配
過濾掉問候語、meta-question、boilerplate
例：「你好」「謝謝」「可以幫我嗎」「請問一下」
成本：$0（純正則，無 AI）
Layer 2 — Prototype Bank
Embedding 向量相似度
預建噪音模式向量庫，用 cosine similarity 比對
例：閒聊、重複確認、格式化請求
成本：$0（Qwen3-8B 本地 Embedding）
Layer 3 — Smart Extraction
LLM 判斷 + 萃取
由 Qwen3:14B（本地）判斷是否值得記住
萃取出結構化的知識事實
成本：$0（Ollama 本地推理）
三層過濾全部在本地運行、零成本 — 只有通過三關的知識才會被向量化存入 LanceDB

Guardrails

零幻覺原則 & 安全邊界

🚫 零幻覺原則 — 每個欄位名都必須有來源
表名、欄位、數值只能來自以下五個來源，禁止猜測、假設、引用「常見做法」
① memory_recall
② bq-api get-table
③ mssql describe
④ Schema .md 檔案
⑤ 用戶提供
🔒 Read-Only 強制
Wrapper 正則攔截 INSERT、UPDATE、DELETE、DROP、ALTER 等全部 DML/DDL
💰 成本確認制
任何 billable query 都要用戶確認，即使 NT$0.01，dry-run ≤ NT$60
🛡 職責邊界
只做資料分析。連線、TLS、驅動問題 → 告知用戶聯繫工程師
Key Insight

跟以往 Agent 的根本差異

傳統 Agent 的 context 沒有向量化，容易撞到 context 上限或語意搜尋不精準，導致表現像失憶。BQ Agent 透過 Qwen3 向量化 + LanceDB hybrid 檢索，自動累積記憶並在下次提問時精準拼湊 Context，達成正確查詢。

Security

資安與環境安全設計

🔒 Gateway 存取控制
綁定：localhost:18789（僅本機可存取）
認證：Token-based（每次請求驗證）
Session：dmScope: per-channel-peer
每個用戶獨立 session，互不干擾
🛡 Exec 指令白名單
所有 exec 經 Gateway 代理，非白名單直接擋
exec-approvals.json：逐條核准可執行指令
追蹤：每條指令記錄最後使用時間與完整命令
Agent 無法直接存取 Shell
🚫 Per-Agent 工具 ACL
BQ Agent 禁用：browser, web_search, process, apply_patch 等 8 項
Per-User 禁用：所有用戶禁 write, edit（防止改檔案）
原則：最小權限，只開放資料分析所需工具
🔑 憑證與資料保護
secrets/ 目錄：chmod 600（僅擁有者可讀）
BQ 成本閘門：NT$60 硬上限 + Read-Only 強制
零幻覺原則：禁止猜測，每個欄位必須有來源
📈 Promptfoo 測試驗證
使用 Promptfoo 對 Agent 進行系統化測試（測試報告：192.168.123.126:15173/eval/）
為什麼選 Promptfoo？開源、支援自定義評估指標、可測試 prompt injection / hallucination / 邊界條件，
能系統化驗證 Agent 在各種場景下是否遵守零幻覺原則、成本管控規則、權限邊界
