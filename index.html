MEMORY

雙層記憶架構

OpenClaw 採用向量記憶 + 檔案記憶雙層設計，兼顧語意搜尋與持久性

🗃
Tier 1 — 向量記憶
LanceDB Pro (MCP Server)
memory_recall(query) // 語意搜尋
memory_store(fact) // 儲存事實
memory_update(id,new) // 更新
memory_forget(id) // 遺忘
用途：Schema 定義、業務規則、語意知識
↔
📄
Tier 2 — 檔案記憶
Workspace 檔案系統
MEMORY.md // 長期策展
memory/YYYY-MM-DD.md // 日誌
SOUL.md / USER.md // 固定
用途：Session 日誌、策展知識、人格設定
向量記憶負責「快速想起相關知識」• 檔案記憶負責「完整保留上下文」
兩者互補，構成 Agent 的完整認知系統

CORE RELATIONSHIP

記憶 × 模型 × Agent 的三角關係

🤖 Agent（人格 + 工具 + 權限）
SOUL.md 定義行為 • AGENTS.md 定義操作 • 獨立 Workspace 隔離
🧠 Model（大腦）
接收 Agent 人格 + 工具指令 + 記憶上下文 → 產出回應與決策
recall
store
🗃 Memory（記憶）
LanceDB 向量記憶（語意搜尋）+ 檔案記憶（持久上下文）
Ollama Embedding 將文字轉為向量存入 LanceDB
🔄 讀取流（Recall）
Agent 發問 → Ollama 向量化問題 → LanceDB 搜尋相似記憶 → 注入 LLM 上下文 → 產出精準回答
💾 寫入流（Store）
LLM 判斷需要記住 → 呼叫 memory_store → Ollama 向量化 → 存入 LanceDB → 未來可被 recall
LIFECYCLE

Session 完整生命週期

從使用者發送訊息到收到回覆的完整資料流

① 使用者訊息（Telegram / CLI）
② Gateway 路由 → 匹配 Agent → 載入 Workspace
③ Context 組裝：SOUL + AGENTS + USER + MEMORY.md + 當日日誌
④ LLM 推理 + memory_recall（查向量記憶）+ MCP 工具呼叫
⑤ 回應產出 + memory_store（存新知識）+ 更新日誌
⑥ 訊息送達（Telegram 回傳 + 附件 PNG / CSV）

SECURITY

安全與隔離設計

🔒 Workspace 隔離
每個 Agent 獨立 Workspace
記憶不共享 • 檔案不互通
防止 Agent 間資訊外洩
🛡 工具權限 ACL
Base Profile + Deny List per Agent
BQ Research 禁用 browser、write
PM Tracker 禁用 edit、apply_patch
💰 成本管控

Dry-run 預估成本再執行
Wrapper 層面強制攔截
🔑 憑證管理
所有 API Key 存於 secrets/
檔案權限 600（僅擁有者可讀）
Telegram Token 獨立分離
